# 校园论坛AI审核服务实现文档

## 1. 系统架构

### 1.1 整体架构

```
SpringBoot后端 → Flask RESTful API → 审核引擎 → 情感分析模型
                                      ↓
                                敏感词过滤器
```

### 1.2 核心模块

- **Flask RESTful API**：提供HTTP接口，处理审核请求
- **审核引擎**：综合敏感词检测和情感分析结果，做出审核决策
- **敏感词过滤器**：使用DFA算法检测敏感词
- **情感分析模型**：基于LSTM的情感分类模型

## 2. 功能实现

### 2.1 敏感词过滤

#### 实现原理

- 使用DFA（确定有限自动机）算法实现高效敏感词检测
- 构建敏感词字典树，实现O(n)时间复杂度的检测
- 支持多类别敏感词库管理

#### 核心功能

- 多类别敏感词检测
- 敏感词严重程度评估
- 词库热更新
- 敏感词定位和标记

### 2.2 情感分析

#### 实现原理

- 使用LSTM（长短期记忆网络）模型进行情感分类
- 支持三分类：正向、负向、中性
- 基于PyTorch框架实现

#### 核心功能

- 文本情感分析
- 情感概率计算
- 模型推理优化
- 批量处理支持

### 2.3 审核决策引擎

#### 实现原理

- 综合敏感词检测结果和情感分析结果
- 基于规则的决策系统
- 支持不同内容类型的差异化处理

#### 核心功能

- 多维度风险评估
- 智能审核决策
- 审核策略配置

## 3. 技术实现

### 3.1 敏感词过滤模块

#### 核心文件

- `sensitive_words.py`：实现DFA敏感词过滤器

#### 关键类

- `WordCategory`：敏感词类别枚举
- `MatchResult`：敏感词匹配结果
- `DFAFilter`：DFA敏感词过滤器
- `SensitiveWordManager`：敏感词库管理器

#### 关键方法

- `add_word()`：添加敏感词到DFA树
- `parse()`：检测文本中的敏感词
- `load_from_directory()`：从目录加载敏感词库
- `reload_vocabulary()`：重新加载词库

### 3.2 情感分析模块

#### 核心文件

- `model.py`：LSTM模型定义
- `inference.py`：模型推理实现
- `preprocess.py`：文本预处理

#### 关键类

- `SentimentLSTM`：LSTM情感分类模型
- `SentimentInference`：情感分析推理器

#### 关键方法

- `predict()`：预测文本情感
- `predict_batch()`：批量预测文本情感
- `preprocess_text()`：文本预处理
- `load_model()`：加载模型权重

### 3.3 审核引擎模块

#### 核心文件

- `moderation.py`：审核引擎实现

#### 关键类

- `ModerationAction`：审核动作枚举
- `ContentType`：内容类型枚举
- `ModerationResult`：审核结果
- `ModerationEngine`：审核引擎

#### 关键方法

- `moderate()`：审核单个文本
- `moderate_batch()`：批量审核文本
- `calculate_risk_score()`：计算风险分数
- `make_decision()`：做出审核决策

### 3.4 API服务模块

#### 核心文件

- `app.py`：Flask RESTful API服务

#### 关键接口

- `POST /moderate`：内容审核
- `POST /moderate/batch`：批量审核
- `GET/POST /moderation/config`：审核策略配置
- `POST /sensitive_words/check`：敏感词检测
- `GET /sensitive_words/stats`：词库统计
- `GET /health`：健康检查

## 4. 模型训练

### 4.1 数据集

#### 4.1.1 数据集来源

本项目使用以下数据集进行模型训练：

1. **ChnSentiCorp_htl_all.csv**
   - **来源**：ChnSentiCorp（中文情感分析基准数据集）
   - **描述**：酒店评论情感分析数据集，包含正面、负向、中性三类情感标签
   - **特点**：
     - 标注质量高，是中文情感分析的权威基准数据集
     - 包含丰富的情感表达和语言现象
     - 数据量约10k+条
   - **标签格式**：0=负向，1=正向，2=中性

2. **waimai_10k.csv**
   - **来源**：外卖平台评论数据
   - **描述**：外卖平台用户评价数据集，主要为正面评价
   - **特点**：
     - 与校园论坛评论场景接近（都是服务评价）
     - 语言风格贴近实际用户表达
     - 数据量10k条
   - **标签格式**：0=负向，1=正向（主要为正向评价）

3. **neutral.csv**
   - **来源**：人工标注的中性情感数据
   - **描述**：中性情感评论数据集
   - **特点**：
     - 补充中性样本，平衡数据分布
     - 包含客观描述性文本
   - **标签格式**：2=中性

#### 4.1.2 数据集选择理由

选择上述数据集的原因：

1. **符合开题报告要求**："结合 ChnSentiCorp 等公开数据集与人工标注的校园评论样本"
2. **场景匹配度高**：外卖评论与校园论坛评论场景相似，都是服务评价类内容
3. **数据质量高**：ChnSentiCorp是权威基准数据集，标注质量可靠
4. **数据量充足**：总数据量约20k+条，满足模型训练需求
5. **情感分布均衡**：通过组合不同数据集，实现正负中三类情感的相对平衡

#### 4.1.3 数据集统计

| 数据集                   | 总数      | 正向     | 负向     | 中性     | 用途           |
| ------------------------ | --------- | -------- | -------- | -------- | -------------- |
| ChnSentiCorp_htl_all.csv | ~10k+     | ✓        | ✓        | ✓        | 主要训练集     |
| waimai_10k.csv           | 10k       | ✓        | ✓        | -        | 补充正面数据   |
| neutral.csv              | 未统计    | -        | -        | ✓        | 补充中性数据   |
| **合计**                 | **~20k+** | **丰富** | **丰富** | **丰富** | **完整训练集** |

#### 4.1.4 数据预处理流程

1. **数据合并**：将多个数据集合并为统一格式
2. **标签统一**：统一标签格式（0=负向，1=正向，2=中性）
3. **数据清洗**：
   - 去除重复数据
   - 过滤过短文本（<3个字符）
   - 去除特殊字符和表情符号
4. **数据增强**：对少数类别进行数据增强，平衡各类别样本
5. **数据划分**：训练集/验证集/测试集按8:1:1比例划分

### 4.2 训练流程

1. **数据预处理**：
   - 分词
   - 去停用词
   - 构建词表
   - 序列编码

2. **模型训练**：
   - 定义LSTM模型
   - 设置超参数
   - 训练模型
   - 模型评估

3. **模型优化**：
   - 调整模型参数
   - 防止过拟合
   - 提高模型性能

### 4.3 模型评估

- **准确率**：模型预测正确的比例
- **精确率**：预测为正例且实际为正例的比例
- **召回率**：实际为正例且被预测为正例的比例
- **F1分数**：精确率和召回率的调和平均

## 6. 审核策略配置

### 6.1 敏感词策略

- **严重级别**：根据敏感词类别和严重程度
- **风险分数**：基于敏感词严重程度计算
- **拦截阈值**：敏感词风险分数阈值
- **审核阈值**：需要人工审核的风险分数阈值

### 6.2 情感分析策略

- **负向阈值**：直接拦截的负向情感概率阈值
- **审核阈值**：需要人工审核的负向情感概率阈值

### 6.3 内容类型策略

- **帖子**：标准审核策略
- **评论**：标准审核策略
- **用户名**：严格审核策略
- **签名**：标准审核策略

## 7. 部署指南

- jieba 0.42+


## 8. API接口文档

### 8.1 内容审核接口

#### 请求

- **URL**: `/moderate`
- **方法**: POST
- **参数**:
  - `text` (required): 待审核文本
  - `content_type` (optional): 内容类型 (topic/comment/username/signature)

#### 响应

```json
{
  "success": true,
  "data": {
    "action": "ALLOW",
    "reason": "内容正常",
    "sentiment_score": 0.95,
    "sentiment_label": "positive",
    "sensitive_words": [],
    "risk_score": 0,
    "process_time": 0.01
  }
}
```

### 8.2 批量审核接口

#### 请求

- **URL**: `/moderate/batch`
- **方法**: POST
- **参数**:
  - `items` (required): 待审核项目数组
    - `text`: 待审核文本
    - `content_type`: 内容类型

#### 响应

```json
{
  "success": true,
  "data": [
    {
      "action": "ALLOW",
      "reason": "内容正常",
      "sentiment_score": 0.95,
      "sentiment_label": "positive",
      "sensitive_words": [],
      "risk_score": 0,
      "process_time": 0.01
    }
  ]
}
```

### 8.3 敏感词检测接口

#### 请求

- **URL**: `/sensitive_words/check`
- **方法**: POST
- **参数**:
  - `text` (required): 待检测文本

#### 响应

```json
{
  "success": true,
  "data": {
    "matched": true,
    "words": ["敏感词1", "敏感词2"],
    "categories": ["色情", "广告"],
    "risk_score": 45.0
  }
}
```

### 8.4 健康检查接口

#### 请求

- **URL**: `/health`
- **方法**: GET

#### 响应

```json
{
  "status": "healthy",
  "timestamp": "2024-01-01T12:00:00",
  "model_loaded": true,
  "moderation_loaded": true
}
```
